I Hard Math Problem

说是 困难的数学题，其实算是一个简单的数学结论？可能需要一些数学直觉。

题目大意

在一个无限大的方形平面方格中，每一个格子里可以放 三种物件，标号为H、G、E，也可以什么都不放。对于物品H，周围相邻的位置必须至少放置一个G和一个E，这里相邻的定义是有一条边重合。

问：在无限大的平面中 H的个数除以平面所有方格数量的最大值是多少，保留6位小数。

E Bogo Sort

题目大意

首先输入一个$N (1 \leq N \leq 10^5)$ ，之后输入一个长度为$n$的排列 $p_1,p_2,p_3,…,p_n$。

每一次变换时，把第$p_i$个数放到第$i$位。

问：可以给出多少个不同的长度为N的排列，在经过任意次数的变换之后，变为$1,2,3,…,n$，最后的答案可能很大，要对 $10^N$取模。

思路

场上比较自然的想到了这就是要计算不同置换的个数。

离散数学中当时蜻蜓点水地提到了一点点，我想我需要好好学习一下数学了。

首先可以构成环，假设一个环中数字的数量是$cnt$，这些环中的数在经过$cnt$次变换后会回到原处，可以通过写一个暴力程序非常直观的感受到这一点。

假设构成了$M$个环，环中数个数为 $cnt_1,cnt_2,cnt_3,…,cnt_m$。

那么最后的答案就是 这些数取LCM（最小公倍数）。

虽然的确猜测到了这个$10^N$可能 是虚晃一枪，但是如果给你整连续好多素数的确可能会爆long long，而且当时也不是很确定$10^N$到底是不是虚晃一枪，如果用C++ 大数模板的话，测算最大的数会出问题，所以使用java或python才是更加明智的选择，而且java和python的 IDE可以提供代码补全，所以编写时也没有想象中那么麻烦。

使用不同的语言时，提交的时候一定一定要注意选好对应的编译器！

场上通过的java代码

这个题用java写并不是很困难，要熟练掌握BigInteger的使用技巧啊。

//命名为Main.java程序
import java.io.*;
import java.util.*;
import java.math.* ;


public class Main {
    static int p[]=new int[100010];
    static boolean vis[]= new boolean[100010];

    static int vec[]=new int[100010];
    static int cnt=0;

    static BigInteger gcd(BigInteger a,BigInteger b){
        if(b.equals(BigInteger.ZERO)) return a;
        return gcd(b,a.mod(b));
    }
    static BigInteger lcm(BigInteger a,BigInteger b){
        return a.divide(gcd(a,b)).multiply(b);
    }

    public static void main(String[] args) {

        Scanner cin = new Scanner(System.in);
        int N=cin.nextInt();
        BigInteger mod=BigInteger.TEN;
        mod=mod.pow(N);

        for(int i=1;i<=N;i++){
            p[i]=cin.nextInt();
        //    System.out.println(p[i]);
        }

        cnt=0;

        for(int i=1;i<=N;i++) {
            int t = 0;
            if (vis[i] == false) {

                int aim = i;
                int iter = p[i];
                t++;
                while (iter != aim) {
                    vis[iter]=true;
                    iter = p[iter];
                    t++;
                }
                vec[cnt++] = t;
            }
        }
        BigInteger ans=BigInteger.ONE;
        for(int i=0;i<cnt;i++){
            BigInteger t=BigInteger.valueOf(vec[i]);
            ans=lcm(ans,t);
        }
		ans=ans.mod(mod);
        System.out.println(ans);
        }
    }

D Drop Voicing

核心思路

首先把这个数列放到环上去思考，这个环上有一个指针。

连续进行第二种操作的本质 是移动这个圆环上的指针，

连续进行第一种操作的本质 是把$ p_n $ 向前移动。

枚举数列起点，之后在长度为$n$的数列上寻找最长上升子序列，时间复杂度$O(n^3)$ 刚刚好。

const int maxv=1e3+10;
int dp[maxv];

int n;
int a[maxv];
int num[maxv];

int LIS(int l,int r){
	int n=r-l+1;
	for(int i=1;i<=n;i++){
		num[i]=a[l+i-1];
		dp[i]=1;
	}

	for(int i=1;i<=n;i++)
		for(int j=1;j<i;j++) 
			if(num[j]<num[i]) dp[i]=max(dp[i],dp[j]+1);

	return dp[n];
}

			int t=n-LIS(i,i+n-1);
			ans=min(ans,t);
		}

B Graph

图论的题目，要不然就是比较简单，要不然就是要套一个比较经典的现成算法。

这道题目我个人认为属于后者。

但是这个现成的图论算法，应该掌握理解的啊！

目前只能当做STL，放到我的模板中了。

具体思路

这里直接引用了 官方题解的表述（个人认为非常简洁和清晰

做法:可以发现任意两个点之间连边的权值都是固定的。由于图始终联通，所以两点间始终存在至少一条 路径，如果存在多条，根据环的异或和为0，两点间的路径的异或和应该相等，且始终是固定的。

所以可以给每个点一个权值，那么两点间的连边权值就应该是两端点权的异或。

接下来的问题就是异或最小生成树。

异或和最小生成树问题

https://www.cnblogs.com/zwfymqz/archive/2018/10/14/9786641.html

涉及到了01Trie，以及Boruvka算法（用于求解最小生成树的）。

具体细节看代码注释吧。

//
// n 最大值是1e5
// 0<= a_i <=2^30
// 异或最小生成树 
// 
//
// 调用前需要给定节点 int a[] 的值 下标是1-n
// 需要赋予初始值 res=0
//
// sort(a+1,a+n+1)
// dfs(1,n,29)
//
// 然后输入答案res(类型为long long)

#define LL ll

const int N = 1e5 + 10;
const int inf=0x3f3f3f3f;

int tr[N * 30][2], idx;
int a[N];

LL res = 0;

void insert(int num) {
    int p = 0;
    for(int i = 29; i >= 0; i--) {
        int x = (num >> i) & 1;
        if(!tr[p][x]) tr[p][x] = ++idx;
        p = tr[p][x];
    }
}
 
int query(int num) {
    int res = 0;
    int p = 0;
    for(int i = 29; i >= 0; i--) {
        int x = (num >> i) & 1;
        if(tr[p][x]) {
            p = tr[p][x];
        }
        else {
            p = tr[p][!x];
            res += (1 << i);
        }
    }
    return res;
}
void dfs(int l, int r, int depth) {
    if(depth == -1 || l >= r) return;
    int mid = l - 1;
    while(mid < r && ((a[mid + 1] >> depth) & 1) == 0) mid++;
 
    dfs(l, mid, depth - 1);
    dfs(mid + 1, r, depth - 1);
    if(mid == l - 1 || mid == r) return;
    for(int i = l; i <= mid; i++) {
        insert(a[i]);
    }
    int num = inf;
    for(int i = mid + 1; i <= r; i++) {
        num = min(num, query(a[i]));
    }
    res += (LL)num;
    for(int i = 0; i <= idx; i++) {
        tr[i][1] = tr[i][0] = 0;
    }
    idx = 0;
}

struct edge{
	int to,cost;
	edge(int to=0,int cost=0):to(to),cost(cost){}
};

vector<edge> G[N];
int n;

void addedge(int u,int v,int w){
	G[u].pb(edge(v,w));
	G[v].pb(edge(u,w));
}

void foo(int u,int pre){
	for(int i=0;i<G[u].size();i++){
		edge now = G[u][i];
		int to=now.to;
		int w=now.cost;
		if(to==pre) continue;
		a[to]=a[u]^w;
		foo(to,u);
	}
}

int main(){
	while(~scanf("%d",&n)){
		inc(i,1,n) G[i].clear();
		inc(i,1,(n-1)){
			int u=read();
			int v=read();
			int w=read();
			u++,v++;
			addedge(u,v,w);
		}
		a[1]=0;
		foo(1,0);
		sort(a+1,a+n+1);
		dfs(1,n,29);
		printf("%lld\n",res);
	}
	return 0;
}
